# 6-factor-docs Usage Examples

## Audit a README and docs folder

```text
Use the 6-factor-docs skill to audit this repository documentation for positioning, onboarding, information architecture, examples, accuracy, maintenance, and Agent handoff readiness.
```

## Turn rough docs into release-ready docs

```text
Use the 6-factor-docs skill to refactor this project's README, guides, examples, and release notes so a new user can complete the first useful action in under five minutes.
```

## Pair with product readiness

```text
Use 13-factor-product-readiness as the primary Skill and 6-factor-docs as the secondary Skill to make this repository feel launch-ready and easy to adopt.
```

## Pair with CLI review

```text
Use 11-factor-cli to review command UX, then use 6-factor-docs to make the installation, quickstart, help examples, and troubleshooting docs match the actual CLI behavior.
```
