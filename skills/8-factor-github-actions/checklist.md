# 8-factor-github-actions Checklist

## Project clarity

- [ ] The project has a clear positioning statement.
- [ ] The target user is explicit.
- [ ] The main use case is understandable in under one minute.
- [ ] The README includes a quick start.

## Required artifacts

- [ ] README exists and explains usage.
- [ ] Examples exist and can be copied.
- [ ] Version is tracked.
- [ ] Changelog or release notes exist.
- [ ] Validation commands are documented.

## Domain-specific checks

- [ ] Trigger design
- [ ] Permissions
- [ ] Secrets handling
- [ ] Caching
- [ ] Build/test steps
- [ ] Artifacts
- [ ] Failure messages
- [ ] Reuse and maintainability

## Product readiness

- [ ] Installation or setup path is clear.
- [ ] Failure modes are documented.
- [ ] Logs or errors help users recover.
- [ ] Defaults are reasonable.
- [ ] The project can be validated locally or in CI.

## Agent usability

- [ ] An Agent can identify what files to inspect.
- [ ] An Agent can produce a report from the provided templates.
- [ ] An Agent can produce a concrete refactor plan.
- [ ] The Skill does not depend on hidden context.
