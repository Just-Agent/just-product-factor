# 15-factor-observability Usage Examples

## Agent project

```text
Use the 15-factor-observability skill to audit this Agent project for logs, traces, tool-call evidence, validation artifacts, and debuggability.
```

## RAG system

```text
Use the 15-factor-observability skill with 9-factor-rag to make retrieval failures, citation gaps, chunking issues, and evaluation runs easier to diagnose.
```

## MCP server

```text
Use the 15-factor-observability skill with 7-factor-mcp to review tool invocation logs, permission denials, schema validation failures, and integration traces.
```

## GitHub Actions

```text
Use the 15-factor-observability skill with 8-factor-github-actions to improve workflow summaries, failure logs, release artifacts, and deployment diagnostics.
```

## Security-aware debugging

```text
Use the 15-factor-observability skill with 14-factor-security-privacy to make this app easier to debug without leaking secrets or user data.
```

## Product release

```text
Use the 15-factor-observability skill after 13-factor-product-readiness to ensure the release has validation evidence, troubleshooting notes, and maintainable diagnostic artifacts.
```
